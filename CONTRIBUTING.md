# How To Contribute

We welcome anyone reading this to contribute to our project. Below we describe ways you can help us improve it. Please follow our workflows to make collaboration easier for everyone.

## Quick Considerations

* We follow the **"One Feature. One Branch. One Commit."** rule.
* If someone is assigned to a PR or Issue, please respect that assignment.
* Development by Maintainers usually goes directly to PRs.

## Raising Issues

If you want to inform us about bugs, possible features, or changes:

1. Check if there is already an issue related to your problem.

2. If not, create a new issue.

3. The issue should describe the problem:

   * **Bug reports** should include:

     * the encountered problem
     * steps to reproduce

   * **Feature proposals** should include:

     * a description of the feature scope
     * an explanation of the added value

4. The level of detail required depends on the scope and complexity of the change.

5. If additional details are necessary, Maintainers will ask for them.

6. Maintainers mark an Issue as accepted for development by assigning an ID, e.g. `STW-100`.

Maintainers reserve the right to prioritize or decline issues based on current project goals.

## Contributing Code

If you want to contribute code, check open Issues or [raise a new one](#raising-issues). We suggest starting with issues marked as `good first issue`.

1. Read the Issue. It should:

   * be open
   * not have anyone assigned
   * have an ID assigned by Maintainers

2. Write a comment indicating that you are interested in implementing it.

3. Maintainers will assign you to the Issue.

4. You may then start working on it:

   * clone the repository
   * create a branch with the given ID (e.g. `STW-100`) from the current `main`

5. When ready, create a PR. It should contain **one commit** with a name starting with the ID, e.g. `STW-100 Feature Name`. If you need to make changes, squash them into a single commit.

6. When your code passes automated tests, request a review from **@pankogito**.

7. Address requested changes and re-request a review when ready.

8. Wait for a merge.

Maintainers reserve the right to change assignments or push changes if necessary.

## Communication

Communication between maintainers and external collaborators will take place through comments on Issues and PRs.

If you need to contact us directly, please reach out to **@pankogito**. Information will be passed along if necessary.
